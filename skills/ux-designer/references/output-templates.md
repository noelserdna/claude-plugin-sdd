### Token Naming Conventions

| Pattern | Example | Rule |
|---------|---------|------|
| `{category}.{property}` | `color.primary` | Top-level categorization |
| `{category}.{property}-{variant}` | `color.primary-hover` | State variants with hyphen |
| `{category}.{scale}` | `spacing.4` | Numeric scales |
| `{category}.{size}` | `font.size.lg` | Named sizes |
| `{category}.{semantic}` | `color.error` | Semantic naming preferred |

### Multi-Theme Structure

Colors use `{ "light": value, "dark": value }` objects to support theme switching. Non-color tokens are theme-independent (spacing, typography, etc. remain constant across themes).

### Framework Integration Notes

- **CSS Custom Properties**: Map `color.primary.light` to `--color-primary` in `:root` and `color.primary.dark` to `--color-primary` in `[data-theme="dark"]`
- **Tailwind**: Extend `theme.extend` in `tailwind.config.js` with token values
- **Style Dictionary**: JSON structure is compatible with Style Dictionary transforms
- **CSS-in-JS**: Import JSON directly and access via dot notation
